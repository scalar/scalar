Add .js extensions to declaration file imports for Node.js ESM compatibility

Declaration files generated by vue-tsc and tsc were missing file extensions in their relative imports, causing TypeScript errors with `moduleResolution: "node16"` or `"nodenext"`. This adds `ts-add-js-extension` to the build process for all Vue packages to ensure declaration files have proper `.js` extensions for ESM module resolution.
